### Problem 1

This problem is worth 7% total (3% for part A, 2% for part B, 2% for part C).

a) When a user program executes `TRAP`, the CPU switches from user mode to kernel mode, saves the current PC and processor state (registers, flags) onto the kernel stack, and uses the trap number to index into the interrupt vector table (IVT) to find the address of the corresponding system call handler. The handler executes, then restores the saved state and returns to user mode via a return-from-trap instruction.
- Key concept: the TRAP mechanism for transitioning between user mode and kernel mode, including saving/restoring context and using the interrupt vector table.

b) The OS identifies which system call is being requested via a system call number, typically placed in a designated register (e.g., `$v0` in MIPS or `eax` in x86) before the `TRAP` instruction. The kernel uses this number to index into the system call table.
- Key concept: system call dispatch through a system call number stored in a register and looked up in a system call table.

c) A subroutine `CALL`/`RETURN` does not change the privilege level (stays in user mode), so the called routine cannot access kernel data structures or execute privileged instructions. `TRAP` explicitly transitions to kernel mode, which is essential for security and protection.
- Key concept: privilege level transition — `TRAP` elevates to kernel mode while `CALL` remains in user mode, which is necessary for OS protection.

### Problem 2

This problem is worth 6% total (3% each).

a) **Preemptive SJF** (also called SRTF — Shortest Remaining Time First) can cause starvation: if short jobs keep arriving, a long job's remaining time is never the shortest, so it is perpetually preempted and never completes. **Nonpreemptive SJF** can also starve long jobs if short jobs continuously arrive before the long job is selected.
- Key concept: starvation in scheduling — SJF (both preemptive and nonpreemptive) can indefinitely postpone long jobs when short jobs dominate.

b) **Preemptive FCFS** and **Nonpreemptive FCFS** are essentially the same, because FCFS serves jobs in arrival order and a running job is never preempted (there is no shorter/higher-priority criterion to trigger preemption). Alternatively, **Round-Robin with an infinite time quantum** is equivalent to **nonpreemptive FCFS**.
- Key concept: equivalence between scheduling algorithms when preemption criteria are never triggered or when the time quantum is infinite.

### Problem 4

This problem is worth 6% total (3% each).

a) **Paging has internal fragmentation** (the last page of a process may not be fully used) **but no external fragmentation** (memory is allocated in fixed-size frames, so there are no unusable gaps between allocated regions).
- Key concept: internal vs. external fragmentation — paging eliminates external fragmentation by using fixed-size pages/frames but introduces internal fragmentation in the last page.

b) **Segmentation** is an alternative: it divides a process's address space into variable-size logical segments (code, data, stack). **Advantage**: segments correspond to logical program structure, making sharing and protection more natural. **Disadvantage**: variable-size segments lead to external fragmentation.
- Key concept: segmentation as a virtual memory scheme — logical division with natural sharing but susceptible to external fragmentation.

### Problem 8

This problem is worth 5%.

**Amdahl's Law** states that the overall speedup from improving a fraction $f$ of the computation by a factor $S$ is: $\text{Speedup} = \frac{1}{(1 - f) + f/S}$. It shows that the non-improved (sequential) portion limits the maximum achievable speedup. As $S \to \infty$, the speedup approaches $1/(1-f)$. This guides system design by identifying bottlenecks: focus optimization effort on the largest fraction of execution time for maximum impact.
- Key concept: Amdahl's Law — quantifying the diminishing returns of parallelization due to the sequential fraction of a workload.

### Problem 15

a) For $t=10$: sequential part = $3 \times 10 = 30$, parallelizable part = $6 \times 10^2 = 600$. Total on 1 processor = $630$. With 15 processors: parallel part = $600/15 = 40$, total = $30 + 40 = 70$. Speedup = $630/70 = 9$.
- Key concept: Amdahl's Law applied — computing speedup by dividing the parallelizable portion among processors while the sequential portion remains fixed.

- Key concept: load imbalance reduces parallel speedup — the slowest processor determines the parallel execution time.

### Problem 16

This problem is worth 6% total (2% each).

a) An Omega network for 8 nodes has $\log_2 8 = 3$ stages, each with $8/2 = 4$ switch boxes, for a total of $12$ switches. Connect inputs to outputs using the shuffle-exchange pattern: at each stage, the routing is determined by one bit of the destination address.
- Key concept: Omega network topology — $\log_2 N$ stages of $N/2$ switches each, with shuffle-based interconnections.

b) **Advantage**: Omega network uses only $\frac{N}{2} \log_2 N = 12$ switches (vs. $N^2 = 64$ for crossbar), significantly fewer. **Disadvantage**: Omega network is a blocking network — some simultaneous communication patterns cannot be routed without conflict, while a crossbar is non-blocking.
- Key concept: trade-off between switch count (cost) and blocking behavior (functionality) in interconnection networks.

c) Two practical considerations: (1) **Wire length and signal propagation delay** — longer wires between distant nodes increase latency and may require signal repeaters. (2) **Physical layout and routing congestion** — different link distances complicate PCB/chip layout and can create hotspots or bandwidth bottlenecks.
- Key concept: physical implementation constraints in interconnection networks — wire delay and routing complexity affect real-world performance beyond the abstract switch count.
